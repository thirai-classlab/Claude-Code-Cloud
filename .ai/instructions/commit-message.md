# コミットメッセージに関する指示

このファイルでは、AI アシスタントがコミットメッセージを生成する際に従うべき具体的な指示を定義します。

* コミットメッセージは短く、要点を押さえたものにしてください。
* 変更内容の概要を明確に書いてください。
* 必要に応じてエピック/サブタスクIDを先頭に付与してください（例: `E20250820-001 ST-002: LWC テーブル表示のソート追加`）。
* 慣例: `feat|fix|refactor|test|docs|chore: 短い説明` を推奨します。
* 逐次実行と連携する場合、`[EYYYYMMDD-### ST-###]` を末尾に付与し、必要に応じて `status: in-progress -> done` のような状態遷移を本文に1行で記録します。
